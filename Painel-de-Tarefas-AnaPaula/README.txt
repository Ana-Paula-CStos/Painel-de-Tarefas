1. Qual foi sua lógica para estruturar o projeto?
Baseado um pouco a partir de um projeto antigo que eu mesma fiz, escolhi completar o desafio usando HTML pois é uma linguagem que tenho mais conhecimento e me sinto mais confiante em usar. Claro, ainda estou aprendendo e aperfeiçoando as outras linguagens.
Interface: simples e intuitiva.
Novas tarefas: em formulário, o usuário insere o título da tarefa, descrição, data e nível de prioridade.
Filtros: o usuário consegue filtrar as tarefas por título, status (concluída ou pendente) e organizar por ordem de criação ou prioridade.
Contador de tarefas: permite ao usuário saber quantas tarefas estão pendentes e quantas foram concluídas.
Tarefas: serão exibidas, após serem adicionadas, as tarefas, exibindo título da tarefa, descrição, data de entrega, prioridade e status. Cada tarefa possui botões para alterar status, excluir e adicionar subtarefa, esta última em formato de checklist.
Foi utilizado o localStorage para os dados permaneceren salvos mesmo quando fechar ou atualizar o navegador.
Cada tarefa tem um id único para facilitar edição, exclusão e ordenação.
Ao realizar qualquer ação, a função renderTasks() é chamada para atualizar a interface com os dados mais recentes.

2. Que parte você achou mais difícil ou travou?
Deve ter sido gerenciar múltiplas funcionalidades ao mesmo tempo. Isso exige um bom planejamento da estrutura e da interação entre as funções javascript, fazendo tudo funcionar junto e sem conflito.

3. O que faria diferente se tivesse mais tempo?
Entreguei adiantado, mas sempre estive revisando o código e aperfeiçoando-o.

4. O que faria diferente se fosse para um cliente real?
Gostaria de realizar reuniões de briefing para entender com clareza objetivos, público-alvo e as necessidades reais do cliente, criar documentos importantes como requisitos e manual do usuário.
Seria ótimo também melhorar as funcionalidades, como login e cadastro de funcionários, segurança, calendário de visualização, responsividade.
